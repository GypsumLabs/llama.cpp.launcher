# llama-server 参数说明

```
usage: ./llama-server [options]
```

## 通用选项

| 参数 | 说明 |
|------|------|
| `-h, --help` | **显示帮助信息** — 打印帮助文档并退出 |
| `--version` | **显示版本号** — 打印程序版本并退出 |
| `-m, --model MODEL` | **模型路径** — 指定 GGUF 模型文件的路径 |
| `-mm, --mmproj MODEL` | **多模态投影器路径** — 指定多模态（视觉）投影器文件，用于支持图像输入 |
| `-c, --ctx-size N` | **上下文长度** — 提示词上下文窗口大小，值越大能处理越长的对话，但显存占用越高（默认: 2048） |
| `-n, --n-predict N` | **最大生成长度** — 单次最多生成的 token 数，-1 表示无限制（默认: -1） |

## 采样参数

| 参数 | 说明 |
|------|------|
| `--temp N` | **温度** — 控制输出随机性，值越高越随机/有创意，值越低越确定/保守（默认: 0.7） |
| `--top-p N` | **Top-P 采样** — 核采样，只从累计概率达到 P 的最小 token 集合中采样（默认: 0.9） |
| `--top-k N` | **Top-K 采样** — 只从概率最高的 K 个 token 中采样（默认: 40） |
| `--repeat-penalty N` | **重复惩罚** — 对已出现的 token 施加惩罚以减少重复，1.0 表示不惩罚（默认: 1.1） |
| `--repeat-last-n N` | **惩罚回溯窗口** — 重复惩罚回溯的 token 数量，即检查最近多少个 token 来判断重复（默认: 64） |

## 服务器选项

| 参数 | 说明 |
|------|------|
| `--host HOST` | **监听地址** — 服务器绑定的主机名或 IP 地址，设为 `0.0.0.0` 可从局域网访问（默认: 127.0.0.1） |
| `--port PORT` | **端口** — 服务器监听的端口号（默认: 8080） |
| `--api-key KEY` | **API 密钥** — 设置后客户端请求需携带此密钥进行身份验证（默认: 无） |
| `--threads N` | **线程数** — CPU 推理使用的线程数，-1 表示自动检测（默认: -1） |
| `--batch-size N` | **批处理大小** — 提示词处理（prefill）阶段的批大小，影响首次响应速度（默认: 512） |
| `--ubatch-size N` | **微批处理大小** — 更细粒度的批处理控制（默认: 512） |
| `--parallel N` | **并行序列数** — 最大并发处理的请求数量（默认: 1） |
| `--timeout N` | **超时时间** — 服务器超时，单位秒（默认: 600） |
| `--verbose` | **详细输出** — 启用详细日志输出，方便调试 |
| `--metrics` | **启用监控指标** — 启用 Prometheus 格式的性能监控端点 |
| `--ssl-keyfile FILE` | **SSL 密钥文件** — HTTPS 所需的 SSL 私钥文件路径 |
| `--ssl-certfile FILE` | **SSL 证书文件** — HTTPS 所需的 SSL 证书文件路径 |
| `--ssl-cafile FILE` | **SSL CA 文件** — SSL 证书颁发机构文件路径 |
| `--no-webui` | **禁用 Web 界面** — 关闭内置的 Web 聊天界面 |
| `--no-playground` | **禁用 Playground** — 关闭内置的 API 调试页面 |
| `--no-swagger` | **禁用 Swagger** — 关闭 Swagger API 文档页面 |
| `--no-cors` | **禁用 CORS** — 关闭跨域资源共享，阻止浏览器跨域请求 |

## KV 缓存选项

| 参数 | 说明 |
|------|------|
| `--cache-ram N` | **KV 缓存大小** — KV 缓存的内存上限，单位 MB（默认: auto） |
| `--cache-type-k TYPE` | **K 缓存精度** — Key 缓存的数据类型，可选 f16/q8_0/q4_0 等，低精度省显存但可能影响质量（默认: auto） |
| `--cache-type-v TYPE` | **V 缓存精度** — Value 缓存的数据类型，同上（默认: auto） |
| `--kv-unified` | **统一 KV 缓存** — 将 K 和 V 缓存合并管理 |
| `--fa, --flash-attn` | **启用 Flash Attention** — 使用 Flash Attention 加速推理并降低显存占用 |
| `--no-fa` | **禁用 Flash Attention** — 强制关闭 Flash Attention |

## 多模态选项

| 参数 | 说明 |
|------|------|
| `--image-max-pixels N` | **图像最大像素数** — 输入图像的最大总像素数，超出会被缩放（默认: 16777216，即 4096x4096） |
| `--image-min-pixels N` | **图像最小像素数** — 输入图像的最小总像素数（默认: 1） |

## GPU 加速选项

| 参数 | 说明 |
|------|------|
| `-ngl, --n-gpu-layers N` | **GPU 层数** — 将模型的前 N 层卸载到 GPU 运行，设为很大的值（如 999）可将整个模型放入 GPU（默认: 0） |
| `-mg, --main-gpu ID` | **主 GPU** — 多 GPU 时指定主 GPU 的设备 ID（默认: 0） |
| `-ts, --tensor-split S` | **张量分割比例** — 多 GPU 时指定各 GPU 的负载比例，例如 `3,7` 表示 30%/70%（默认: 均分） |
| `-lowvram, --low-vram` | **低显存模式** — 启用低显存优化，牺牲速度换取更低的显存占用 |
| `-nommap` | **禁用内存映射** — 不使用 mmap 加载模型，改为直接读入内存 |
| `-numa` | **启用 NUMA 优化** — 针对多路 CPU（NUMA 架构）服务器的性能优化 |
